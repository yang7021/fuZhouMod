## 1. 注册与自动化 (AutoAdd)

手动一个一个注册卡片会累死人，咱们在主 Mod 文件里用 `AutoAdd` 一劳永逸。

### 1.1 设置步骤
1. 在主类（如 `BasicMod.java`）实现 `EditCardsSubscriber` 接口。
2. 在 `receiveEditCards` 方法中塞入这段“万能暗号”：

```java
@Override
public void receiveEditCards() {
    new AutoAdd(modID) // 加载你自己的 Mod 内容
        .packageFilter(BaseCard.class) // 扫描这个包里的所有卡片类
        .setDefaultSeen(true) // 默认让它们在图鉴里露脸
        .cards(); // 正式添加！
}
```
> [!TIP]
> 只要你的卡片类放在 `BaseCard` 同一级或子包下，写完就能自动进游戏，不用再回主文件折腾。

## 2. 核心代码结构 (The BaseCard)

杀戮尖塔的卡片继承链是这样的：`BaseCard` → `CustomCard` → `AbstractCard`。咱直接用 `BaseCard`，因为它帮咱们省了很多写升级代码的体力活。

### 2.1 关键信息 (CardStats)
每一张卡片都需要一张“身份证”，咱们用 `CardStats` 来统一管理。

```java
public class MyStrike extends BaseCard {
    public static final String ID = makeID(MyStrike.class.getSimpleName()); // 自动获取类名作为 ID
    
    private static final CardStats info = new CardStats(
        MyCharacter.Meta.CARD_COLOR, // 卡片颜色
        CardType.ATTACK,             // 类型：ATTACK/SKILL/POWER/CURSE/STATUS
        CardRarity.BASIC,            // 稀有度：BASIC/COMMON/UNCOMMON/RARE/SPECIAL
        CardTarget.ENEMY,            // 目标：ENEMY (单体), ALL_ENEMY (全体), SELF (自己), NONE
        1                            // 基础耗能：-1 是 X 费，-2 是不可使用
    );
}
```

### 2.2 构造函数与预设数值
```java
private static final int DAMAGE = 6;      // 基础伤害
private static final int UPG_DAMAGE = 3;  // 升级加多少伤害

public MyStrike() {
    super(ID, info); // 把身份证交给爸爸类
    
    // 一张卡的大部分属性升级都可以直接在这里设定
    setDamage(DAMAGE, UPG_DAMAGE);   // 基础值和升级增加值
    setBlock(BLOCK, UPG_BLOCK);     // 同上
    setMagic(MAGIC, UPG_MAGIC);     // 设置 magicNumber 的基础值和升级值
    setCostUpgrade(0);              // 升级后改变卡牌消耗 (比如从 1 费变 0 费)
    
    tags.add(CardTags.STARTER_STRIKE);
    tags.add(CardTags.STRIKE);
}
```

## 3.3 升级机制详解 (Upgrades)

老大爷，这一块儿是重头戏。升级不仅仅是加点伤害，有时候连名字和描述都会变。

### A. 自动挡：BaseCard 的魔力
在 `BaseCard` 里，你不需要手动写 `upgrade()` 方法。只要你在构造函数里用了 `setDamage(6, 3)`，游戏在卡片升级时会自动帮你执行：
- `baseDamage` 增加 3。
- 自动把卡片标记为“已升级”。
- 自动刷新卡片文本。

**常用“自动挡”函数：**
- `setDamage(base, upgrade)`: 伤害。
- `setBlock(base, upgrade)`: 格挡。
- `setMagic(base, upgrade)`: `magicNumber`。
- `setCostUpgrade(newCost)`: 直接设升级后的费用。
- `setExhaust(false, true)`: 升级后获得“消耗”属性。
- `setInnate(false, true)`: 升级后获得“固有”属性。

### B. 手动挡：重写 upgrade()
如果您想搞点高级的，比如升级后给卡牌加个标签，或者升两次级，那就得覆写 `upgrade()`：

```java
@Override
public void upgrade() {
    if (!upgraded) { // 如果卡片还没升级
        upgradeName(); // 名字变绿
        upgradeDamage(UPG_DAMAGE); // 增加伤害数值
        // 在这里加你自己的特殊逻辑，比如加个标签
        this.tags.add(CardTags.HEALING);
        // 如果在 JSON 里写了 UPGRADE_DESCRIPTION，还得调这一句
        initializeDescription(); 
    }
}
```

### C. 升级后的文本变化
如果在升级后，卡片的描述发生了根本性变化（比如原来是“抽 1 张牌”，升级后变成“抽 1 张牌并消耗”），你需要在 `CardStrings.json` 里提供 `UPGRADE_DESCRIPTION`：

```json
"${modID}:YourCard": {
  "NAME": "狂暴",
  "DESCRIPTION": "抽 !M! 张牌。",
  "UPGRADE_DESCRIPTION": "抽 !M! 张牌。 NL 消耗。"
}
```
`BaseCard` 只要发现你有 `UPGRADE_DESCRIPTION`，在升级后就会自动帮您切过去！

## 3. 卡牌运行逻辑 (Mechanics)

### 3.1 变量大揭秘
代码里有三兄弟变量最常用：
- **`damage`**: 伤害值（会自动计算力量、易伤等加成）。
- **`block`**: 格挡值（会自动计算敏捷、虚弱等加成）。
- **`magicNumber`**: 万能数值（抽牌数、加特技等，不受游戏内置属性加成影响）。

在文本描述里，它们分别用 `!D!`, `!B!`, `!M!` 来占位。

### 3.2 `use` 方法：让卡牌动起来
当玩家打出卡片，游戏会调用这个方法。我们要往“动作队列”里塞指令：

```java
@Override
public void use(AbstractPlayer p, AbstractMonster m) {
    // addToBot 是把动作塞到队列末尾，按顺序执行
    addToBot(new DamageAction(m, new DamageInfo(p, damage, DamageInfo.DamageType.NORMAL), 
             AbstractGameAction.AttackEffect.SLASH_VERTICAL));
}
```
- **NORMAL**: 普通伤害（可被格挡）。
- **THORNS**: 荆棘伤害（不可被力量加成）。
- **HP_LOSS**: 直接扣血（无视格挡）。

## 4. 本地化文本 (Localization)

老大爷，这也是个容易崩溃的点！卡片名和描述得写在 JSON 里。

- **路径**: `resources/yourmodID/localization/eng (或 zh-hans)/CardStrings.json`
- **格式**:
```json
"${modID}:MyStrike": {
  "NAME": "狂暴打击",
  "DESCRIPTION": "造成 !D! 点伤害。",
  "UPGRADE_DESCRIPTION": "哪怕升了级，这个也会自动用新描述替换掉原来的。"
}
```

### 4.1 文本格式化暗号
在 `DESCRIPTION` 里的那些花哨颜色都是这么来的：
- **`#y`**: 关键词变黄金色（如 `#y力量`）。
- **`#b`**: 数字变蓝色。
- **`*`**: 高亮（如 `*符咒`）。
- **`[E]`**: 这是一个炫酷的图标，代表一点**能量**。
- **`NL`**: 强制换行（前后都要留空格，像这样 ` NL `）。

## 5. 美術資源 (Art)

卡片图不是随便一张就能贴的，得按规矩来：

1. **大图 (`_p.png`)**: 尺寸 **500x380**。当玩家右键看卡片详情时显示。
2. **小图 (`.png`)**: 尺寸 **250x190**。在手牌和抽牌堆里显示。

### 5.1 存放路径规范
`resources/yourmodID/images/cards/` 下面分三类：
- `attack/`
- `skill/`
- `power/`
文件名必须和卡片 ID（不带前缀的那部分）完全一致。

## 6. 进阶小贴士 (Polish)

- **升级设置**: 用 `BaseCard` 提供的 `setExhaust(false, true)` 等方法，可以直接定义“升级后增加消耗/获得消耗”。
- **效果顺序**: 如果一张卡既加格挡又打伤害，**必须先给格挡**！不然你打有荆棘的怪时，会先掉血再加格挡，亏死啦。
- **描述顺序**: 习惯上把 `保留`、`固有` 写在开头，`消耗` 写在最后，逻辑顺着来。
