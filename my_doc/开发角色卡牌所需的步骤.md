# 创建杀戮尖塔角色的步骤与美术资源需求

老大爷，根据您提供的BasicMod文档，我为您整理了创建一个杀戮尖塔角色需要完成的开发步骤，以及您需要准备的美术资源清单。

## 一、 开发步骤说明

1. **创建角色类 (The Character Class)**
   - 在项目的 `java/yourmod/character` 目录下创建角色的主类。
   - 设定角色的基础属性：最大生命值(MAX_HP)、每回合能量(ENERGY_PER_TURN)、初始金币(STARTING_GOLD)、初始抽牌数(CARD_DRAW)等。

2. **配置本地化文本 (Localization)**
   - 在 `CharacterStrings.json` 文件中配置角色的名称、描述等文本信息，确保角色ID（如"CharacterID"）与代码中对应。

3. **定义相关的枚举 (Enums)**
   - 定义角色职业枚举（`PlayerClass`）。
   - 定义专属卡牌颜色枚举（`AbstractCard.CardColor` 和 `CardLibrary.LibraryType`），必须保持一致。

4. **注册角色图像路径**
   - 在类中声明角色选择界面的按钮、立绘图片，以及卡牌背景图、能量球等图片的存放路径。

5. **编写核心方法**
   - `getStartingDeck()`: 配置角色的初始牌组。这里需要至少准备几张卡牌的ID。
   - `getStartingRelic()`: 配置角色的初始遗物。

6. **在主Mod类中注册 (Adding the Character)**
   - 在主Mod类的 `initialize()` 方法中，调用 `registerColor()` 注册卡牌颜色。
   - 让主Mod类实现 `EditCharactersSubscriber` 接口。
   - 重写 `receiveEditCharacters()` 方法，并在里面调用 `registerCharacter()` 将角色正式注册进游戏。

7. **开发专属卡牌 (Adding Cards)**
   - **配置自动注册 (AutoAdd)**: 在主Mod类的 `receiveEditCards()` 方法中，使用 `AutoAdd` 扫描并自动注册位于 `BaseCard` 同包或子包下的所有卡牌类。
   - **创建卡牌类**: 为每一张新卡牌创建一个继承自 `BaseCard` (其继承自 `AbstractCard`) 的类。
   - **配置卡牌基础信息**: 在卡牌类中定义 ID (如 `makeID("Strike")`)、并在构造方法 `super(ID, info)` 中传入包含 颜色(Color)、类型(Attack/Skill/Power)、稀有度(Rarity)、目标(Target)和费用(Cost) 的 `CardStats` 对象。
   - **设置卡牌数值与升级**: 在构造方法中使用 `setDamage()`, `setBlock()`, `setMagic()` 等方法配置基础数值与升级提升的数值。
   - **分配卡牌标签 (Tags)**: 例如为打击牌添加 `tags.add(CardTags.STARTER_STRIKE)` 和 `tags.add(CardTags.STRIKE)`。
   - **实现卡牌效果 (use方法)**: 重写 `use(AbstractPlayer p, AbstractMonster m)` 方法，通过调用 `addToBot(...)` 向动作队列添加具体效果动作，如 `DamageAction` 等。

8. **后续完善**
   - 补充基础卡池：角色至少需要75张卡牌（包含足够的普通、罕见、稀有卡牌，以及不同类型的攻击、技能、能力卡）以避免游戏在进入卡牌奖励或商店时崩溃。
   - 补充专属的遗物（包括初始遗物升级版、无能量Boss遗物和能量Boss遗物等）。
   - 定制视觉效果，例如修改默认的能量球实现（`CustomOrb`）、修改人物动画和碎心结局CG等。

## 二、 【重点】需准备的美术资源清单 (Images)

为了让角色视觉完整，您需要准备以下美术图片，通常存放在 `resources/yourmod/images/character/` 目录下。

### 1. 角色相关图像
| 资源用途 | 说明 | 尺寸建议/格式 |
| -- | -- | -- |
| **选人界面按钮** (`select/button.png`) | 游戏开始界面，选择该角色的圆形图标 | 114x114 |
| **选人界面立绘** (`select/portrait.png`) | 选择该角色时背景显示的大图 | 1920x1200 |
| **角色战斗贴图** (`image.png`) | 战斗中角色的静态显示图（如果不使用骨骼动画）| 根据需求 |
| **角色肩部图1** (`shoulder.png`) | 在篝火休息时的贴图 | 根据需求 |
| **角色肩部图2** (`shoulder2.png`) | 在篝火休息时的另一种边缘贴图 | 根据需求 |
| **角色死亡图** (`corpse.png`) | 战斗失败后倒地的贴图 | 根据需求 |

### 2. 卡牌背景 (Card Backgrounds) - 【通用的卡牌底板相框】
卡牌分为小视图(标准牌)和大型立绘视图(单张检视时显示)，总共需要使用6张底板图（*诅咒和状态牌通常默认使用技能牌的背景*）。
**【注】这6张底板图属于通用的相框资源！如果您没有画师，完全可以直接提取原版战士的红色相框背景图，或者使用别人做好的开源修仙/基础模板图，直接复制粘贴到您的美术目录里覆盖即可，不需要专门请人重画！**
| 卡牌类型 | 标准小图 (`base`) | 检视大图 (`_p`) |
| -- | -- | -- |
| **攻击牌背** | `cardback/bg_attack.png` | `cardback/bg_attack_p.png` |
| **技能牌背** | `cardback/bg_skill.png` | `cardback/bg_skill_p.png` |
| **能力牌背** | `cardback/bg_power.png` | `cardback/bg_power_p.png` |

### 3. 卡牌能量球 (Card Energy Orbs)
| 资源用途 | 说明 |
| -- | -- |
| `cardback/energy_orb.png` | 绘制在卡牌左上角的费用图标（小图） |
| `cardback/energy_orb_p.png` | 绘制在卡牌左上角的费用图标（大致检视时候的大图） |
| `cardback/small_orb.png` | 文本描述中用来代表能量的微型图标（如“获得1点[E]”） |

### 4. 战斗大能量球 (Combat CustomOrb) - 【非必须】
显示在战斗界面左下角的能量槽大球。如果您不想自己画，**完全可以在代码中复用原版角色的能量球（例如原版战士的红球，代码里传 `new EnergyOrbRed()` 即可）**。
如果您决定自定义，则需要准备：
* **多层部件图阵列**：由多张图片叠加组成（推荐128x128），通过代码控制每一层的相互旋转。
* **获得能量特效** (`energyorb/vfx.png`)：当获得能量时在球表面扩展并淡出的闪烁特效图（推荐256x256）。

### 5. 碎心结局CG (Heart Kill Cutscene) - 可选
如果您希望打通心脏后有您角色特有的通关CG，取代默认战士的碎片画面：
* **碎片CG组图**：尺寸为 1920x1200 的图片组成的序列，用来填满过场动画屏幕。

### 6. 每张具体卡牌的插画 (Card Portraits)
除开通用卡背和能量球，**您创建的每一张新卡牌，都需要一张专属的插画**：
* **标准插画图** (`cards/your_card_name.png`)：显示在普通小视图卡牌中央的贴图 (推荐 250x190)。
* **检视大插画图** (`cards/your_card_name_p.png`)：在图鉴或右键检视卡牌时显示的高清大图 (推荐 500x380)。
